🔒 Security Scan

FindingSeverityDetailsRecommendation
unsafe-inline in CSP Medium 7 files use 'unsafe-inline' for style-src. Catalog UI also uses it for script-src. Use nonce-based style loading. Already nonce-based for scripts in main panels.
child_process usage Low 3 files import child_process: securityManager.ts (execFile for SSL), aiKitManager.ts (execFile for git), mcpHandlers.ts (execFileSync for port check). All use execFile/execFileSync (safer than exec). ✅ Good practice.
Hardcoded secrets None No hardcoded passwords or API keys found in source. ✅ Using SecretStorage for API tokens. Good.
RegExp from user input Low 6 new RegExp() usages. Most use controlled patterns (ID prefixes, known patterns). Validate intakeManager.ts L310 — field pattern from config may need sanitization.
MCP HTTP Auth Medium HTTP transport supports bearer/apikey auth. Auth is optional by default. Document that production MCP should enable auth + SSL. Add warning on startup if auth disabled.
Connect-src policy Low CSP allows http://localhost:* and ws://localhost:* for local MCP. Acceptable for local development. Consider restricting port range.

📈 Pitch Deck

🎯 Problem

Developers context-switch between IDE, browser-based project tools, and AI assistants — losing flow state and productivity.

💡 Solution

AI Command Center brings planning, AI integration, and service orchestration directly into VS Code — zero context switching.

🌍 Market

TAM: 73M developers worldwide. SAM: 30M VS Code users. SOM: 2M AI-forward dev teams.

📊 Traction

395 AI resources (22 agents, 49 skills, 274 instructions, 49 prompts). 90 commands. 5 integrations. Active development with v2.0.16.

🏆 Moat

Deepest AI-ley kit ecosystem. MCP-first architecture. Growing instruction/skill library with community contribution model.

💰 Model

Open Core: Free MIT-licensed extension + Premium skills/integrations + Enterprise features (SSO, audit, SLA).
